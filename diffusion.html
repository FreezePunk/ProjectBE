Diffusion des résultats et valorisation du projet

Pour donner de la visibilité à notre travail, nous avons mis en place ce site web et utilisé GitHub Pages pour le publier facilement et gratuitement.

Dès le début du projet, la question de la diffusion des résultats a été centrale. Nous avons souhaité rendre notre démarche et nos analyses accessibles non seulement à l’équipe pédagogique, mais aussi à toute personne intéressée par l’intelligence artificielle appliquée aux échecs.

Pourquoi un site web ?

Le choix de créer un site web s’est imposé : il permet de regrouper toutes les étapes, résultats et analyses de notre projet sous une forme claire et interactive. Chacun peut naviguer entre les différentes sections, consulter nos choix techniques, nos résultats, nos difficultés, et pourquoi pas s’en inspirer pour d’autres projets.

GitHub Pages : héberger gratuitement le site

Pour publier le site, nous avons utilisé GitHub Pages, un service gratuit permettant d’héberger des sites statiques directement à partir d’un dépôt GitHub. Ce choix présente plusieurs avantages :

  • Mise en ligne ultra simple : il suffit de pousser les fichiers HTML/CSS/JS dans une branche dédiée du dépôt GitHub (main ou gh-pages),
  • Accès public immédiat via une URL fournie par GitHub,
  • Aucun besoin de serveur ni de configuration complexe,
  • Maintenance et mises à jour facilitées (un simple git push suffit à déployer une nouvelle version du site).

Comment ça marche ?

1. Nous avons créé un dépôt GitHub contenant tous les fichiers de notre site.
2. Dans les paramètres du dépôt, nous avons activé GitHub Pages et choisi la branche à publier.
3. Après chaque modification, un git add ., git commit puis git push met le site à jour en ligne.

Toute personne peut désormais consulter nos résultats et notre démarche à l’adresse publique du site.

Ce mode de diffusion nous a permis de valoriser notre travail, de le rendre accessible à tous, et d’illustrer concrètement notre projet.
